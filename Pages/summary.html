Purpose

The goal of this study is to demonstrate ten real-world network problems across all OSI layers, then simulate a working solution for each. Every guide is written from an Admin Perspective, providing CLI snippets, verification steps, roll-back plans and Packet Tracer files when possible.

What We Built

  • 02. EtherChannel (LACP) – Link Aggregation
  • 03. HSRP – Gateway Redundancy
  • 04. Wi-Fi 7 Channel Overlap (6 GHz)
  • 05. TCP Window-Scaling Mismatch
  • 06. SMB Signing Mismatch – Session Setup Fail
  • 07. Port Security – MAC Lockdown
  • 08. DHCP Snooping – Rogue Server Block
  • 09. HTTP/2 Downgrade – Missing ALPN
  • 10. STP Core & BPDU Guard Edge

Key Takeaways

  1. Layered thinking works. Solving issues at the appropriate OSI layer reduces complexity and avoids side-effects.
  2. Simulation ≠ Reality. Tools like CPT are excellent for L2/L3 concepts, but higher-layer protocols (TLS, SMB, HTTP/2) must be tested on real stacks.
  3. Document everything. A concise admin guide speeds up hand-over and future troubleshooting.
