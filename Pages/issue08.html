DHCP Snooping ‚Äì Rogue DHCP Server Blocking

Layer: Data-Link (L2)

Symptom: Clients occasionally receive wrong IP (192.168.3.x) and lose network access

Root Cause: An unauthorized DHCP server on SW2 offers bogus leases; SW1 ports were unprotected

Objective

  • Allow DHCP offers only from the trusted server (192.168.3.1)
  • Drop/Log all rogue DHCP messages from untrusted ports
  • Build lease binding table for future security features (DAI, IPSG)

Baseline Topology

‚Ä¢ Authorized DHCP: Server-PT @ SW2 Fa0/1 ‚Üí 192.168.3.101-200
‚Ä¢ Unauthorized: Server-PT (rogue) @ SW2 Fa0/2 ‚Üí 192.168.3.202
‚Ä¢ Clients PC0-PC3 on SW1 Fa0/1-3
‚Ä¢ SW1 ‚Üî SW2 uplink = Fa0/24 (dotted in the diagram)

Configuration ‚Äì SW1 (Access Switch)

! enable feature
ip dhcp snooping
ip dhcp snooping vlan 3

! mark uplink to SW2 as trusted
interface Fa0/24
 description LINK-TO-SW2
 ip dhcp snooping trust

! all PC access ports remain untrusted (default)
interface range Fa0/1 - 3
 switchport mode access
 switchport access vlan 3

! optional: limit DHCP rate to stop starvation attacks
ip dhcp snooping limit rate 30

Configuration ‚Äì SW2 (Distribution / DHCP segment)

ip dhcp snooping
ip dhcp snooping vlan 3

! Authorized server port is trusted
interface Fa0/1
 description AUTH-DHCP
 ip dhcp snooping trust

! Rogue server port stays untrusted
interface Fa0/2
 description ROGUE-DHCP
 ! (no trust)

! Uplink back to SW1 also trusted
interface Fa0/24
 ip dhcp snooping trust

Verification

  1. show ip dhcp snooping ‚Äì both switches list VLAN 3, trusted ports as expected.
  2. show ip dhcp snooping binding ‚Äì PC0-PC3 receive 192.168.3.101-103 from the authorized server.
  3. Rogue server sends Offer ‚Üí Switch drops; show ip dhcp snooping statistics increments ‚Äúdrops: untrusted‚Äù.
  4. Pings from PC0 to 192.168.3.1 succeed; PCs never get 192.168.3.202 gateway.
