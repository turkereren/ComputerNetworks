Port Security ‚Äì Unauthorized Device Shutdown

Layer: Data-Link (L2)

Symptom: Rogue laptop plugged into access switch causes CAM table flood; legitimate PCs intermittently lose connectivity

Root Cause: Switch ports were open ‚Äì any MAC address could send traffic; attacker ran a MAC-flood tool

Objective

  • Lock down each access port to a single, known MAC
  • Automatically err-disable the port on violation
  • Persist learned MACs across reboots (sticky)

Design Overview

‚Ä¢ Access Switch: Cisco Catalyst 2960-24TT (SW1)
‚Ä¢ PCs PC0-PC14 connected Fa0/1 ‚Äì Fa0/14
‚Ä¢ Port security enforced on all user ports; uplinks excluded

  2. Plug a rogue laptop into Fa0/5 ‚Üí port goes err-disabled within 1 s.
  3. show interface status err-disabled ‚Äì port appears with sec-violation reason.
  4. Legitimate PCs on other ports remain unaffected.
